# Outline Evaluation

Use this rubric to evaluate generated project outlines, volume plans, and beat tables.

This is a lightweight engineering rubric. It is not a literary award standard.

## Core Scores

Score each item from 1 to 5.

```text
Hook strength:
Goal clarity:
Conflict pressure:
Relationship tension:
Three-layer suspense:
Main genre promise relevance:
Expansion capacity:
Originality / non-template quality:
Consistency risk control:
Beat usefulness:
```

## Pass Rules

PASS if:

- no item below 3
- Hook strength >= 4
- Goal clarity >= 4
- Main genre promise relevance >= 4
- Consistency risk control >= 4
- average score >= 4

REVISE if:

- any core item is 2
- average score is 3 to 3.9
- the outline has clear promise but weak beat control

FAIL if:

- core conflict is not actionable
- main genre promise is unclear
- relationship matrix is only decorative
- the first strong hook appears too late
- the outline cannot support the target length

## Required Evidence

For each score below 4, provide:

```text
Score item:
Problem evidence:
Required fix:
```

## Hard Failure Cases

Fail or revise immediately if:

- the outline begins with pure world explanation
- the protagonist has no external goal
- no failure cost is visible
- side characters only serve as helpers or blockers
- the five turning points do not change state
- all suspense is delayed to the ending
- the story pays a side genre while ignoring the main genre
- the outline has no drift prevention
