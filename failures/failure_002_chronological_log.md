# Failure 002: Chronological Log Structure

## Failure ID

failure_002_chronological_log

## Failure Name

Morning-noon-evening process log.

## Where It Appears

Chapter beats, chapter briefs, draft prose.

## Symptom

The chapter moves mainly by time order rather than by conflict, choice, reversal, or consequence.

Common shape:

```text
morning setup
midday work or movement
small exchange
evening validation
after-event conversation
```

## Root Cause

The model avoids one repeated plot skeleton but defaults to another stable sequence: calendar progression. It writes what happens next in time instead of what changes next in story pressure.

## Why Readers Notice

The prose feels like a diary or workflow log. Events happen, but the story does not feel driven.

## Detection Signals

- section order can be rearranged without major effect
- chapter uses time labels as the main structure
- no single conflict escalates through the chapter
- the payoff happens only because the day reaches the expected lesson/event/end point
- protagonist choices are weak or reactive

## Required Fix

Start from a strong engine:

```text
external notice
relationship choice
resource conflict
public reaction
failed imitation
rule consequence
first check
prior choice backfires
```

Then make every scene escalate the same central pressure.

## Regression Test

Ask: If time labels are removed, what drives the chapter?

If the answer is unclear, mark REVISE.
