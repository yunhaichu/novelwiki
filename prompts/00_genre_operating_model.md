# Genre Operating Model Prompt

Use this prompt after `prompts/00_genre_mode_contract.md` and before `prompts/00_reality_causal_preflight.md`, major conflict design, protagonist growth design, and chapter planning.

The purpose is to build a runnable model of how the genre world operates before generating plot events.

This prompt is upstream of chapter design. It prevents the model from inventing dramatic events from genre tropes without proving the world's resource, hierarchy, body, ability, survival, and opportunity logic.

## Core Principle

```text
Do not generate events first.
Build the operating world first, then let events grow from it.
```

A genre operating model answers:

- how people survive;
- how resources circulate;
- how hierarchy works;
- how capability grows;
- how organizations benefit;
- where low-level characters can find a narrow opportunity;
- which trope-like events are impossible or too early.

## Required Inputs

Read:

- user premise;
- genre mode contract;
- selected reference settings;
- base settings if already drafted;
- user constraints;
- any approved canon if the novel already exists.

## Traditional Xianxia Operating Model Requirements

For traditional xianxia, explicitly define:

- what spirit roots measure;
- what counts as qi sense, qi drawing, and qi entering the body;
- why orthodox sect methods differ from rogue or village fragments;
- what low-level cultivation consumes: food, qi, sleep, body strength, time, spirit, medicine, or environment;
- how sects classify outer disciples, named disciples, servants, trial workers, rogue candidates, and rejected candidates;
- why a sect would keep a weak candidate instead of rejecting them;
- how spirit rice, coarse grain, tail spring water, herb residue, furnace ash, scrap talismans, and discarded objects differ;
- why low-level labor is oppressive but still survivable;
- where the first usable crack can appear without becoming a full treasure;
- what a weak goldfinger can notice before it can change reality.

## Checks

Mark `REVISE` if:

- the model lists genre tropes but does not define operations;
- resource flow is unclear;
- hierarchy costs and benefits are unclear;
- body / capability growth has no cost;
- low-level survival is impossible;
- the protagonist's first opportunity is a gift rather than a crack in the system;
- special ability is too broad, too early, or has no limitation;
- social behavior ignores hierarchy;
- opening chapter constraints still allow a full goldfinger reveal or major-faction overreaction.
