# Character Action Logic Prompt

Use this prompt when creating or revising recurring character files, story bible character entries, or chapter briefs involving important decisions.

The goal is to make characters act from desire, fear, flaw, and decision pattern rather than from author convenience.

## Required Inputs

Read:

- approved type contract if present
- project outline
- volume plan if present
- story bible if present
- existing character file if present
- relationship map if present
- relevant chapter beat or brief if this is chapter-specific

## Output Object

For each major or recurring character, produce:

```text
Character:
External desire:
Inner need:
Core fear:
Dominant flaw:
Flaw trigger:
Default protection priority:
- life / status / relationship / principle / control / truth / belonging / freedom
Information threshold:
- acts on instinct / waits for evidence / avoids decision / over-verifies
Conflict mode:
- compete / avoid / compromise / submit / cooperate / manipulate
Self-justification:
First wrong method:
How this flaw creates plot:
Arc direction:
Current knowledge boundary:
Current relationship pressure:
What this character will not say directly:
Stress action:
```

## Chapter-Specific Add-On

When used for a chapter brief, add:

```text
Why must this character act now?
What does this character protect first in this scene?
Which desire, fear, flaw, or pressure is triggered?
What wrong or partial choice can this produce?
What consequence does that choice create?
What would make this character change method?
```

## Rules

- Do not write static labels only.
- A flaw must be triggerable.
- A desire must create visible action.
- A fear must change decisions.
- A decision pattern must be stable enough to recognize across scenes.
- A character can change only when pressure, evidence, relationship, or cost forces change.
- Do not let characters say their entire emotional thesis directly unless the scene has earned it.

## Failure Patterns

Mark these as risks if present:

```text
character has background but no current desire
character has flaw but flaw creates no cost
character acts only because the plot needs it
character changes behavior without trigger
character dialogue explains theme instead of pursuing pressure
side character exists only to make protagonist look clever, calm, or funny
```

## Output

Output only the character action logic artifact.
