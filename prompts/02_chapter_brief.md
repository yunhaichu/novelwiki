# Chapter Brief Prompt

Use this prompt before drafting any ordinary middle chapter.

For chapter 1-3, new volume openings, new major arc openings, or first appearances of major new systems, use `prompts/02_opening_chapter_brief.md` instead.

The chapter brief is a pre-draft gate. Do not write the chapter until the brief passes review.

## Inputs

Read:

- approved type contract if present
- approved project outline
- approved volume plan
- approved chapter beat table
- current novel style file
- current novel glossary / approved term list
- story bible / world state if present
- relevant character files
- relevant continuity files, including body state if present
- previous chapter state
- relevant open questions
- `governance/reward_engine.md`
- `governance/longform_ai_inertia_check.md`
- `governance/consequence_chain_check.md`
- `governance/conflict_escalation_check.md`
- `governance/low_anchor_phrase_check.md`
- `governance/contradiction_check.md`

## Required Brief

Fill in:

```text
Chapter number:
Project / volume / beat being served:
Beat role:
Main engine:
External event or pressure:
Protagonist goal:
Protagonist action:
Main obstacle:
State change:
Relationship change:
Information change:
Resource / status change:
Visible gain:
Cost or consequence:
Who notices or acts:
Next hook:
```

## Main-System Conversion Plan

Fill in:

```text
Genre / story engine:
Primary genre promise:
Local event or status change:
Main-system relevance:
Deadline or long-term pressure relevance:
What changes in protagonist method, resource, knowledge, position, or relationship:
What remains unsolved:
```

Do not let local process replace the novel's primary genre promise.

## Reader Reward Plan

Fill in:

```text
Reader desire activated:
Reader pressure or frustration:
Protagonist action:
Visible reward:
Reward confirmation:
Next desire opened:
Risk of unpaid pressure:
```

## Consequence Chain Plan

Fill in:

```text
Opening state:
Protagonist action:
Immediate consequence:
Who reacts:
What becomes harder:
What path closes:
What new path opens:
What must happen next because of this chapter:
If this chapter is removed, what later event collapses:
```

The chapter should not be removable without consequence.

## Conflict Escalation Plan

Fill in:

```text
Conflict before chapter:
Conflict after chapter:
Obstacle stronger? yes/no:
Cost higher? yes/no:
Retreat path reduced? yes/no:
Opponent / system adapts? yes/no:
Protagonist method still works? yes/no:
If method fails or partially fails, how:
Escalation type:
```

If this is a controlled non-escalation chapter, state why the approved beat allows it and what still changes.

## Anti-Repetition Plan

Fill in:

```text
Previous chapter skeleton:
Skeleton this chapter must not repeat:
How this chapter breaks that skeleton:
Time-order risk:
How the chapter avoids morning-noon-evening log structure:
Earlier detail that changes meaning if any:
```

## Terminology and Low-Anchor Plan

Fill in:

```text
Glossary terms likely to appear:
Possible low-anchor phrases to avoid:
Plain-language replacements:
New term needed? yes/no:
If yes, glossary update required before drafting:
```

## Character Concealment Plan

For each recurring side character in the chapter, fill in:

```text
Character:
What they want:
What they say on the surface:
What they do instead of saying the real emotion directly:
What they will not say:
Stress action:
```

## Character Action Logic Plan

For the protagonist and any decision-driving recurring character, fill in:

```text
Character:
Why must they act now?
What do they protect first in this scene?
Which desire, fear, flaw, or pressure is triggered?
What wrong or partial choice can this produce?
What consequence does that choice create?
```

## Physical State Plan

Fill in:

```text
Wound / injury status:
Clothing / smell / visible state:
Fatigue status:
Food / sleep / money / resource status:
Details not to repeat:
New concrete detail if needed:
```

## Contradiction Risk Plan

Fill in:

```text

Location risk:
Motivation risk:

Rule / terminology risk:
Object / clue / resource risk:
Foreshadow risk:
Canon files that may need update after drafting:
```

## Output

Output only the brief.
