# Project Viability Gate

Use this gate after high-level premise setup and before formal wiki bootstrap.

Purpose: prevent weak concepts from entering the formal wiki just because the process can run. A project must prove that it can sustain long-form webnovel continuation before `prompts/00_wiki_bootstrap.md` is allowed.

Fast Trial sketches may be used to test desire, but they are non-canon and do not bypass this gate.

## Core Rule

```text
No Project Viability Gate pass, no formal wiki bootstrap.
```

A viable project must have:

- a long-form irreversible trend;
- a protagonist final form stronger than `good person / witness / helper`;
- a three-chapter retention path;
- type-appropriate reader returns;
- reusable growth assets, not only mystery escalation.

## Required Inputs

Read:

- novel spine / premise;
- genre mode contract if available;
- genre operating model if available;
- cosmic civilization arena if macro story;
- Earth civilization value if Earth-entry story;
- unified power logic if multi-system story;
- modern Chinese entry bridge if modern-entry story;
- irreversible trend anchor draft;
- protagonist growth track draft;
- Reader Hook / Payoff Ladder draft;
- Fast Trial sketch if available.

## Hard Failure Rules

Mark `REVISE` if:

- the irreversible trend only supports single-unit cases;
- the story can continue only by adding unrelated incidents;
- the protagonist final form is only `good person`, `witness`, `helper`, `observer`, or `survivor`;
- the first three chapters mainly roll mysteries forward without growth assets, status movement, relationship movement, or concrete problem-solving capacity;
- reader return is not a type-webnovel return and is only `interesting setting`, `mystery`, or `deep theme`;
- chapter one relies on forced coincidence, forced face-slapping, or psychologically false reactions;
- the project has no clear volume-one goal.

Mark `KILL` if:

- the premise has no long-form engine after revision attempts;
- the protagonist has no meaningful final-form path;
- the story's main attraction is only one twist or one mystery;
- the project cannot produce a concrete reason to continue after chapter three.

## Pass Rule

Mark `ALLOW` only if:

- the project can support long-form escalation;
- the protagonist has a strong final-form path;
- the first three chapters create concrete retention;
- early chapters create reusable growth assets or stage pressure;
- reader return matches genre expectation;
- the opening can be written without forced plot devices.
