# Generic Common Reference

Use this reference when a project does not clearly fit a specialized genre reference or when it intentionally blends genres.

This file is a reference setting base. It is not canon for any novel.

## Function

The generic reference should help a project establish a minimal usable story base without forcing a specific genre system.

It is suitable for:

- original fantasy
- mixed genre
- soft speculative fiction
- slice-of-life with special premise
- projects whose exact genre is still being clarified

## Required Base Questions

Before drafting prose, answer only what the opening and first volume need:

```text
What social or physical world does the protagonist live in?
What can the protagonist normally access?
What is hard to access?
What institution, group, family, place, or rule creates pressure?
What resource matters early?
What object or document can change the protagonist's options?
What does the protagonist know?
What must remain unknown?
What kind of payoff does the genre promise?
```

## Identity And Authority

Every major role should have:

- visible function
- authority or lack of authority
- dependence on others
- what they can block
- what they can offer
- what they fear losing

Do not use a role label as a complete character.

## Organization And Process

Any organization should have:

- purpose
- gatekeeping method
- record or memory method
- reward method
- punishment or exclusion method
- weakness or blind spot

Any process should have:

- entry condition
- visible step
- decision point
- outcome
- trace left behind

## Resource And Object

Any important resource should have:

- holder
- source
- use
- scarcity
- cost
- risk
- possible consequence if lost

Any important object should change action options. If it does not change action, relationship, information, status, or risk, it should not be treated as a major setting item.

## Space And Location

A useful location should shape action through:

- access
- danger
- witness
- route
- concealment
- resource
- authority
- social meaning

Avoid locations that are only mood labels.

## Knowledge Boundary

Separate:

- what the reader knows
- what the protagonist knows
- what allies know
- what opponents know
- what institutions can verify
- what remains unknown

Do not let the narration confirm secrets early unless the project deliberately uses omniscient narration.

## Genre Guard

Avoid:

- importing genre mechanics not declared by the project.
- creating world rules that do not change choices.
- using vague labels such as ancient rule, hidden order, old power, system, fate, or bloodline without concrete effect.
- replacing protagonist pressure with setting explanation.
- building a complete encyclopedia before reader entry.
