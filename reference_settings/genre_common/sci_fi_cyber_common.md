# Science Fiction / Cyber Common Reference

Use this reference for science fiction, cyberpunk, future technology, artificial intelligence, algorithmic systems, data economies, engineered society, platform control, and space-technology projects.

This file is a reference setting base. It is not canon for any novel.

## Identity And Authority

Common roles:

- ordinary resident: lives under platform, city, corporate, or technical-system rules.
- low-level technician: repairs devices, interfaces, equipment, or local failures; lacks high permissions.
- hacker: attacks, bypasses, traces, hides, or spoofs system behavior under risk of detection.
- security engineer: maintains audit logs, access control, incident response, and risk review.
- platform administrator: controls account, permission, content, trade, identity, or access within system limits.
- data broker: buys, sells, filters, or hides data, identity, or leads; trust and traceability create risk.
- corporate employee / director: executes or controls projects, data, security, acquisition, and suppression.
- law / regulatory agent: investigates, freezes, blocks, arrests, audits, or orders disclosure within jurisdiction.
- mercenary / security contractor: provides outsourced force under contract.
- AI agent / synthetic proxy: acts within objective, access, data, and permission constraints.
- researcher: develops algorithm, material, energy, biology, cognition, or interface systems under funding and ethics constraints.
- scavenger / repairer: recovers old equipment, data fragments, hardware, or forbidden tools.

Authority rule:

Technology does not remove authority boundaries. Every meaningful action should depend on access, credential, hardware, data, energy, permission, exploit, jurisdiction, or trace risk.

## Organization And Process

Common organizations:

- megacorporation: resource monopoly, infrastructure, employment, surveillance, pressure.
- city administration: public order, identity, district control, emergency commands.
- data exchange: data trading, identity, evidence, and risk.
- augmentation / biotech / robotics company: body or capability technology.
- security contractor: outsourced force, monitoring, containment.
- black market network: illegal equipment, data, medicine, identity, passage.
- AI regulator / protocol authority: algorithm control, ethics, compliance, shutdown rights.
- research institute: experimental system, anomaly, ethics conflict.
- underground resistance: counter-system alliance and betrayal risk.
- media network: exposure, rumor, manipulation, public pressure.

Common processes:

- identity authentication: needs account, biometrics, credential, token, or proof chain.
- permission request: needs identity, task, superior authorization, system path, or exploit.
- system intrusion: needs vulnerability, computing power, time window, access point, or insider help.
- log tracing: needs record, timestamp, access chain, sensor, or audit permission.
- augmentation installation: needs device, surgeon, money, compatibility, license, and maintenance.
- data transaction: needs data asset, buyer, encryption, verification, and delivery risk.
- area lockdown: needs authority, system command, security presence, and escape consequence.
- drone patrol / automated enforcement: needs route, recognition rule, energy, and response protocol.
- account freeze: needs rule, log, complaint, algorithm flag, or human approval.
- energy rationing: needs account, rank, district, infrastructure, or political pressure.

Process rule:

A technical process should produce a visible state change: access granted or denied, trace left, alarm triggered, identity changed, resource locked, route closed, reputation damaged, or new leverage gained.

## Resource And Object

Useful technology resources:

- account / credential / identity token / biometric signature
- access log / audit trail / sensor record / camera segment
- algorithm model / training data / black-box score
- exploit / patch / malware / firewall rule
- chip / implant / prosthetic / drone / robot
- battery / reactor credit / compute quota / network bandwidth
- encrypted file / key / hardware wallet / data shard
- maintenance tool / old device / forbidden interface
- city map / restricted-zone permit / transport pass

Resource rule:

A technology resource should have compatibility, access condition, traceability, failure mode, and owner or controller. Avoid technology that solves every conflict without cost or detection.

## Space And Location

Common spaces:

- platform backend, server room, data center, corporate tower, lower district, black market clinic, repair shop, city checkpoint, transit station, restricted lab, drone corridor, surveillance blind spot, orbital dock, quarantine lab.

Space rule:

Space should be shaped by sensors, access gates, network coverage, patrol patterns, energy availability, class boundary, physical bottleneck, and trace risk.

## Social Life

Science-fiction social pressure often comes from:

- identity score or access rank
- algorithmic classification
- debt to platform, company, or medical system
- employment dependency
- body modification cost
- surveillance and self-censorship
- data ownership and privacy loss
- district segregation
- automation replacing human judgment

Dialogue should not become technical documentation. Characters usually speak through operational need, fear of trace, status pressure, bargaining, sarcasm, concealment, or urgent protocol.

## Genre Guard

Avoid:

- using vague terms such as algorithm, quantum, neural, cyber, protocol, or AI without a visible effect.
- letting a hacker bypass everything without time, risk, access, or countermeasure.
- making AI omniscient unless the story defines its sensors and limits.
- replacing human choice with system output.
- adding technology labels that do not change action, risk, access, or consequence.
- using surveillance only as atmosphere rather than pressure, evidence, or constraint.
- importing magical power logic without a declared genre blend.
