# Emergent Plot Workflow

This document defines the repository's generic workflow for creating stronger long-form plot through controlled emergence.

The goal is not to ask an LLM to invent events freely. The goal is to design a conflict system that can generate plot through constrained reactions.

## Core Formula

```text
Plot emergence = interest conflict × resource limits × information asymmetry × bounded cognition × costly choice × consequence chain
```

## Three-Level Emergence

Long-form plot should operate on three levels with different frequencies.

```text
State-level / major faction emergence: rare, provides large direction.
Organization-level emergence: occasional, converts large pressure into local action.
Character-level emergence: frequent, should happen every chapter and often every scene.
```

### State-Level Emergence

State-level forces include countries, empires, sect alliances, corporations, ruling houses, planetary regimes, or any force that can shape the macro conflict.

Use them to define why the world moves.

Do not make state-level actors directly solve every chapter.

### Organization-Level Emergence

Organization-level forces include sects, companies, bureaus, clans, guilds, merchant houses, military commands, schools, gangs, temples, labs, or local offices.

Use them to transfer state-level pressure into concrete friction: records, permits, orders, resources, locations, witnesses, rituals, payments, access, punishment.

### Character-Level Emergence

Characters carry the plot in the chapter.

Each important character must have personal desire, fear, limited knowledge, misjudgment, available resources, and something they cannot afford to lose.

Characters should not act as all-knowing representatives of a faction.

## Controlled Emergence

Emergence must be controlled by direction, not by pre-deciding every event.

Use these controls:

1. **Attractors**
   Define the state the story is moving toward, such as: a neutral bureau collapses, a protagonist becomes a key evidence holder, a hidden old case links to the current crisis, or a faction is forced to publicly choose a side.

2. **State Variables**
   Track changes after each chapter, such as protagonist leverage, organization suspicion, faction conflict heat, public trust, evidence strength, relationship trust, access level, and reader-known truth.

3. **Pressure Injection Order**
   Decide what kind of pressure enters the arena at each stage: record pressure, resource pressure, person pressure, public-face pressure, time pressure, false-cooperation pressure.

4. **Reaction Rules**
   Define how each organization or character tends to react under pressure.

5. **Phase Gates**
   Do not let macro conflict escalate too early. Early chapters should use character and organization pressure. State-level action should appear as documents, orders, rumors, prices, troop movement, or delayed institutional effects unless the volume phase allows direct macro escalation.

6. **Protagonist Advantage Ledger**
   The protagonist can pay costs, but each chapter should give at least one new usable advantage: evidence, information, relationship, access, identity, enemy misjudgment, operational space, or insight.

7. **Reader Reward Ledger**
   Each chapter should answer or partially answer something, not only add new mysteries.

## Recommended Workflow

```text
project scope / type promise
-> base settings
-> major conflict engine
-> dramatic arena
-> actor cognition cards
-> emergence control plan
-> reader entry gate
-> emergent chapter design
-> scene simulation
-> draft
-> emergent plot review
-> chapter state update
```

## Chapter-Level Rule

Every chapter should answer these questions:

```text
1. Which major conflict is pressing from below?
2. Which organization converts that pressure into concrete action?
3. Which character is forced into a costly choice?
4. What does the protagonist gain, despite the cost?
5. What does the reader receive as reward?
6. Which organization or character must react next?
```

If a chapter produces only danger but no protagonist advantage, revise it.

If a chapter produces only a personal incident with no organization or macro consequence, revise it.

If a chapter only explains faction politics but forces no character choice, revise it.

## Frequency Control

A normal chapter should prioritize character-level emergence.

```text
Macro pressure: usually implicit, one line / one object / one order / one consequence.
Organization pressure: visible, one major institutional action or obstacle.
Character pressure: frequent, several choices, misjudgments, refusals, concealments, bargains.
```

Avoid making major factions directly appear in every chapter.

Major factions should feel like gravity, not like constant speaking characters.

## Good Chapter Outcome

The best chapter ending is usually:

```text
The protagonist is in more danger,
but now has a more useful piece of leverage.
```

Examples of useful leverage:

- a document;
- a witness name;
- a contradictory record;
- a location route;
- a debt owed by another character;
- proof that one organization is lying;
- an enemy's mistaken belief;
- access to a new scene;
- a forced relationship with a stronger actor.

## Failure Modes

Mark the design as weak if:

- the LLM invents random events instead of deriving them from the arena;
- all actors seem to know the whole truth;
- factions act only because the plot needs them to;
- the protagonist only suffers and gains no usable asset;
- the chapter adds mystery but answers nothing;
- a major state-level actor solves a problem that should be handled by local organizations and characters;
- organizations are only labels and do not control resources, records, access, money, violence, rituals, or legitimacy.
