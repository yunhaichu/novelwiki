State movement types:
- `leverage_gained` — protagonist gains new ability/access/information
- `leverage_lost` — protagonist loses something useful
- `assumption_failed` — old understanding proven wrong
- `relationship_shift` — character relationship changes state
- `pressure_advance` — clock moves, stake increases
- `question_answered` — a prior debt or question resolved
- `new_conflict` — introduces a fresh problem or pressure
- `preparation` — protagonist sets up for future move
- `aftermath` — deals with consequences of previous events
- `world_reveal` — reader learns something new about the world

## Hard Checks (run after each chapter sync)

1. No 3 consecutive chapters may have zero state movement.
2. No 2 consecutive important chapters may only add mystery without payoff.
3. No repeated hook type may appear 3 times without structural upgrade.
4. Breather chapter budget: have we used more than 30%?
5. Oldest P0 debt: is it >= 3 chapters old?
6. Oldest P1 debt: is it >= 6 chapters old?
7. Any mini-turn in the last 3 chapters? (at least one required)

If any check fails, the next chapter MUST include a structural correction.

## Volume Midpoint Check

After ~50% of chapters are completed:

- State movement vs state stagnation ratio: (movement chapters) / (total chapters so far)
- Target: movement ratio >= 0.6 (at least 60% of chapters produce genuine state change)
- Reader debt: total active / total resolved / total abandoned
- Breather ratio: (breather chapters used) / (total chapters so far)
- Structural upgrades: count of distinct hook/scene types used
- Verdict: ON TRACK / AT RISK / NEEDS REVISION
- If AT RISK or NEEDS REVISION: specify which debt, which rhythm pattern must change
