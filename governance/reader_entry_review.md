# Reader Entry Review Gate

Use this after creating or revising a reader entry gate and before drafting an opening chapter.

The reader entry gate checks whether a first-time target reader can enter the story without reading planning notes or the wiki.

## Decision

Return one of:

```text
PASS
REVISE
```

## Required Checks

### 1. Protagonist Entry

A first-time reader should be able to identify the protagonist quickly.

Mark `REVISE` if the protagonist appears mainly as a job function, setting observer, or process executor.

### 2. Immediate Trouble

The opening must present a concrete immediate trouble, imbalance, pressure, abnormality, danger, or desire gap.

Mark `REVISE` if the opening begins with setting, history, world explanation, or ordinary routine before the trouble becomes legible.

### 3. Immediate Desire

The protagonist must want something now.

Mark `REVISE` if the protagonist only reacts, observes, processes work, or carries backstory without an immediate desire.

### 4. Cost of Inaction

The reader must understand what can be lost soon.

Mark `REVISE` if the opening only creates curiosity without stake, cost, pressure, or consequence.

### 5. Type Promise Visibility

The primary genre promise must be visible on the page.

Mark `REVISE` if the genre promise is only implied by wiki context or long-term outline.

### 6. Information Load

The opening should limit names, terms, systems, rules, mysteries, and backstory.

Mark `REVISE` if the reader must assemble too many elements before knowing what to care about.

### 7. One-Sentence Recap

The gate must include a one-sentence reader recap target.

Mark `REVISE` if the recap target is vague, abstract, overloaded, or impossible to say in ordinary language.

### 8. Concrete Memory Point

The opening should identify at least one concrete memory point: action, object, danger, choice, cost, relationship reaction, or abnormal evidence.

Mark `REVISE` if the memory point is only mood, slogan, theme, or setting atmosphere.
