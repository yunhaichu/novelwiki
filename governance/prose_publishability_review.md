# Prose Publishability Review

Use this after structural chapter review.

This gate separates workflow success from publishable prose.

A chapter can pass structure and still require line revision.

## Decisions

Return both decisions:

```text
Workflow decision: PASS / REVISE
Prose decision: PUBLISHABLE / NEEDS LINE REVISION / STRUCTURAL REWRITE
```

## Workflow Decision

Use the workflow decision for structural gates:

- reader entry;
- type promise;
- protagonist desire;
- consequence chain;
- conflict escalation;
- character action logic;
- contradiction risk.

## Prose Decision

Use the prose decision for line-level and scene-level quality:

- scene naturalness;
- dialogue pressure;
- in-world naming;
- low-anchor phrase drift;
- anti-AI novel phrasing;
- ending concreteness;
- character voice fit;
- artificial humor.

## Decision Rules

### PUBLISHABLE

Use only when:

- workflow decision is PASS;
- prose has no severe artificial diction;
- dialogue fits pressure and character status;
- names feel natural in the world;
- ending lands on concrete action, object, cost, choice, or changed relationship;
- no key payoff depends on an abstract or polished line.

### NEEDS LINE REVISION

Use when:

- structure passes;
- reader entry passes;
- but prose contains artificial terms, overly polished summary, forced jokes, unnatural dialogue, or scene labels.

### STRUCTURAL REWRITE

Use when:

- reader entry fails;
- type promise is invisible;
- protagonist has no immediate desire;
- consequence chain fails;
- conflict does not escalate;
- the chapter is removable.
