# Dialogue Pressure Check

Use this during drafting, review, and line revision.

This check prevents dialogue from becoming exposition, author commentary, repeated joke setup, or reader-question prompting.

## Core Rule

Dialogue should come from scene pressure.

A line should usually do at least one of these:

- push for a practical goal;
- hide a motive;
- threaten or bargain;
- mark status;
- reveal misunderstanding;
- delay a decision;
- test another character;
- cover fear;
- change relationship state.

## Dialogue Classes

### 1. Pressure Dialogue

Valid dialogue that changes the scene:

```text
Do this.
Don't do that.
Give me this.
I won't sign.
You go first.
Say that again.
```

### 2. Relationship Dialogue

Valid dialogue that changes or reveals a relationship:

```text
You remembered that?
You didn't tell me.
Since when do you speak for us?
```

### 3. Exposition Dialogue

Usually revise if the line mainly explains world, plot, rule, theme, or genre to the reader.

### 4. Joke-Setup Dialogue

Usually revise if a side character exists mainly to set up the protagonist's clever reply or a modern joke.

### 5. Reader-Question Dialogue

Usually revise if a character asks exactly what the reader needs explained but has no scene reason to ask.

## Checks

For each important exchange, ask:

```text
What does each speaker want right now?
What does each speaker fear right now?
What would happen if this line were removed?
Does the line change pressure, information, relationship, status, or cost?
Could this line be shorter and more practical?
```

## Review Rules

Mark `REVISE` if:

- dialogue mainly explains the setting;
- recurring side characters become response machines;
- a character says their complete emotional thesis directly;
- a joke pauses the scene without changing pressure;
- a line is too polished for the speaker's status and situation;
- the same setup-response rhythm repeats across adjacent scenes.

## Output Block For Review

```text

- Lines that change pressure:
- Lines that only explain:
- Lines that feel too polished:
- Side-character function risk:
- Required cuts or rewrites:
```
